# Interior Design Studio — Automation Kit

Automation for a 16-stage interior design + build workflow: lead intake, briefing, survey, proposal/quotation, RAB, procurement, production, handover, after-sales.

Generated by the **cowork-automation-generator**. One repo, three ways to run the
same automations: inside **Cowork**, from a **local Python CLI**, or via a
**Next.js + Convex webapp** where each user brings their own Anthropic API key (BYOK).

## What it automates

| Tool | Purpose |
| --- | --- |
| `read_document` | Read the full text contents of a file in the workspace |
| `list_workspace` | List files in the workspace, optionally filtered by a glob (e.g |
| `write_deliverable` | Write a finished deliverable to ./output (brief, proposal, quotation, RAB, BAST, digest) |
| `save_record` | Append a structured record to the local JSONL datastore (one file per table) |
| `lookup_record` | Search the local datastore for records in a table whose fields contain the query string |

| `draft_proposal` | Draft a project proposal + rough quotation from an approved brief and scope: scope of work, design fee, build estimate, timeline, payment terms, included revisions |

| `generate_bast` | Generate a handover document (Berita Acara Serah Terima) from project data: items delivered, scope completed, warranty terms, and sign-off blocks for studio + client |

### Workflows
- **intake-lead** — on-demand: Stage 1: parse a new inbound inquiry, log the lead, draft a WhatsApp acknowledgment, and create a survey-scheduling task.
- **prepare-briefing** — on-demand: Stage 2: turn briefing notes into a structured design brief and set up the next steps.
- **proposal-pack** — on-demand: Stage 4 + 10: from an approved brief, draft a proposal + rough RAB for human review (approval-gated send).
- **daily-project-digest** — scheduled `0 1 * * *`: Every morning: summarize active projects by stage, flag deadlines at risk and pending approval gates, and create follow-up tasks.

## Best practices for this field
- **Money & client sends are approval-gated** — The agent drafts proposals, quotations, invoices, and WhatsApp replies into ./output; a human reviews and sends. Never auto-send or auto-charge.
- **Survey first** — Never draft a RAB, layout, or 3D direction without the structured survey measurements. Read the survey record/notes first; if missing, create a task to capture them.
- **Respect the 16 stage gates** — Proposal, concept, layout, 3D, and RAB advance only on client approval; contract and handover on payment. Don't mark a stage done without a recorded sign-off.
- **Per-project scoped folder** — Point Cowork at one client/project folder (./inbox + ./output), not your whole drive. Keep each project's photos, notes, and deliverables together.
- **Keep the RAB columnar** — Store BoQ line items as JSONL/CSV records (save_record), not merged-cell spreadsheets. Reconcile category subtotals and the grand total in a verification pass before sending.

Suggested Cowork connectors: whatsapp, google-drive, google-calendar, gmail.

## Run it locally (Python CLI)

```bash
cd local
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -e .
cp .env.example .env        # add your ANTHROPIC_API_KEY
automation doctor           # checks key, config, and tool registry
automation run "Summarize everything in ./inbox and list follow-ups"
automation workflow intake-lead
```

Pick the engine with `--engine direct` (default; plain Anthropic SDK loop) or
`--engine agent` (Claude Agent SDK). Add `--stream` to stream tokens.

## Run it as a webapp (Next.js + Convex, BYOK)

```bash
cd web
npm install
npx convex dev        # creates convex/_generated and sets NEXT_PUBLIC_CONVEX_URL
npm run dev           # http://localhost:3000
```

Open the app, paste your Anthropic API key (stored per session in Convex, never
logged), type a task, and watch the agent work with a live transcript. See
`web/README.md` for the streaming details and the production security note.

## Use it inside Cowork

Copy `.cowork/skills/interior-design-ops` into your Cowork skills folder (or this
whole repo into a folder you grant Cowork access to). See `docs/cowork-setup.md`.

## How it fits together

`automation.config.json` is the single source of truth — tool names, schemas,
system prompt, and workflows. The Python registry (`local/automation/tools.py`)
and the TypeScript dispatcher (`web/lib/tools.ts`) both implement the same tool
handlers, so behavior stays consistent across every surface. Domain-specific
tools were stubbed for you to implement (look for `TODO` in those two files).
